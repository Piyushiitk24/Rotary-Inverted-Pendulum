\chapter{Conclusion and Future Work}

\section{Conclusion}
This project developed an upright-only control stack for a custom stepper-driven rotary (Furuta) inverted pendulum:
\begin{itemize}
  \item A first-principles model was derived and specialized for the physical rig (Chapter~\ref{ch:modelling}).
  \item A practical firmware pipeline was implemented, including derivative filtering, wrap-safe angle handling, glitch rejection, engage logic, soft limits, stall diagnostics, and controlled disarm behavior (Chapters~\ref{ch:measurement}--\ref{ch:actuation}).
  \item Three upright controllers were implemented on the same hardware: linear full-state feedback, hybrid sliding mode control, and full-surface sliding mode control (Chapters~\ref{ch:lin}--\ref{ch:smc4}).
  \item Commanded base-position moves while balancing were demonstrated with the linear controller through nudge mode (Chapter~\ref{ch:results}).
\end{itemize}

Across the pinned seven-run dataset, the linear controller was the most consistent overall choice on this rig. It gave the calmest hold behavior and remained upright in the pinned tap run. The hybrid SMC reduced peak upright excursions well in the pinned hold run, but its disturbed recovery was less reliable in the pinned tap run. The full-surface SMC also balanced upright in undisturbed conditions, but it was the most sensitive to actuator limits and protective shaping on this stepper-driven platform.

The nudge experiment showed that commanded base motion while balancing is practical on the real rig, but the pinned run should be read carefully: it demonstrates successful motion without a fall, not tight settling inside the fixed $\pm\SI{1}{\degree}$ band.

These conclusions are qualitative because each condition is represented by one pinned run only. Stronger claims would require repeated trials for each controller and experiment type.

\section{Future work}
The main next steps suggested by this thesis are:
\begin{itemize}
  \item \textbf{Repeat trials for stronger comparisons.} Run many trials per controller and per experiment type so future reports can show distributions instead of single representative runs.
  \item \textbf{Add explicit tap markers.} A button, trigger, or external sensor would make disturbance onset measurable and would allow cleaner recovery-time metrics.
  \item \textbf{Increase logging bandwidth.} Logging selected signals closer to the \SI{200}{Hz} control rate would improve effort and timing analysis.
  \item \textbf{Use more actuator-aware nonlinear control.} The tap results show that actuator limits matter strongly on this platform. Future nonlinear designs should account for bounded actuation more directly.
  \item \textbf{Improve state estimation.} Better derivative estimation or observer-based methods could reduce noise sensitivity and false aborts.
\end{itemize}
